## 2025 — 초기 설계 결정

### 배포 플랫폼: Netlify → Vercel로 변경
Netlify 무료 플랜의 크레딧 기반 과금 구조가 이 프로젝트의 사용 패턴과 맞지 않았습니다.
5초 polling 기준 월 392 크레딧이 필요한데 무료 플랜은 300 크레딧으로 초과합니다.
Vercel은 리소스 기반 과금이라 월 4회, 6시간, 20명 규모에서 무료 플랜으로 충분히 운영 가능합니다.

### 프레임워크: Vite React → Next.js 16 App Router로 변경
Vercel 선택 후 API Routes(Route Handlers)를 자연스럽게 쓰려면 Next.js가 가장 적합합니다.
별도 서버리스 함수 설정 없이 `src/app/api/` 경로만으로 엔드포인트를 구성할 수 있습니다.

### GAS 완전 제거 결정
초기에는 GAS를 API 서버로 유지하는 안을 검토했으나,
GAS의 `google.script.run` 방식은 외부 HTTP 호출과 호환되지 않아 `doGet`/`doPost` 전면 재작성이 필요했습니다.
재작성 공수가 크고 장기적으로 기술부채가 되므로 Vercel Route Handlers로 완전 대체하기로 결정했습니다.

### 실시간 동기화: Polling 30초 간격 채택
Google Sheets는 WebSocket을 지원하지 않습니다.
5초 polling은 Vercel 무료 플랜에서도 사용 가능하지만, 출석부 특성상 초단위 동기화가 불필요하고
30초 polling + Optimistic Update 조합으로 사용자 경험과 서버 비용 모두 만족할 수 있습니다.

### 출석 상태 저장 방식: 레코드 유무로 판단
GAS 원본 코드 분석 결과, Attendance 시트에 `"출석"` 레코드가 존재하면 출석,
없으면 결석으로 간주하는 구조입니다. 별도 `"결석"` 값은 저장하지 않습니다.
마이그레이션 후에도 이 규칙을 동일하게 유지합니다 (기존 데이터 호환성).

### 관리자 인증: 세션 기반 간이 인증
별도 OAuth나 JWT 도입 없이 비밀번호 모달 → `POST /api/auth` 검증 → `sessionStorage` 토큰 저장 방식을 채택합니다.
공개 접근 서비스이고 관리자가 소수(교역자 수준)이므로 이 정도 보안 수준이 적합합니다.
비밀번호는 Vercel 환경변수(`ADMIN_PASSWORD`)에 보관합니다.
